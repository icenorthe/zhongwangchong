# 平安夜支付接口演示程序

本项目是基于平安夜支付接口开发的PHP演示程序，用于展示如何在PHP项目中集成平安夜支付功能。

## 目录结构

```
qjpay_demo/
├── config.php                # 配置文件
├── index.php                 # 演示首页
├── notify.php                # 异步通知处理
├── return.php                # 同步跳转处理
├── documentation.php         # 接口文档
├── clear_session.php         # 清除临时配置
├── libs/                     # 工具库
│   └── functions.php         # 公共函数库
├── examples/                 # 示例代码
│   ├── create_order.php      # API创建订单示例
│   ├── submit_order.php      # 跳转支付示例
│   ├── query_order.php       # 订单查询示例
│   └── query_user.php        # 用户信息查询示例
└── logs/                     # 日志目录
```

## 安装说明

1. 将整个`qjpay_demo`目录上传到您的Web服务器
2. 确保PHP版本 >= 7.0
3. 确保`logs`目录具有写入权限
4. 修改`config.php`中的配置信息（见下文）

## 配置说明

在使用本演示程序前，需要在`config.php`文件中进行如下配置：

1. 修改商户信息
   ```php
   'merchant' => [
       'pid' => '您的商户ID',
       'key' => '您的商户密钥',
   ],
   ```

2. 修改回调地址
   ```php
   'callback' => [
       'notify_url' => 'http://您的域名/qjpay_demo/notify.php',
       'return_url' => 'http://您的域名/qjpay_demo/return.php',
   ],
   ```

3. 其他配置项根据需要调整

## 临时配置功能

本演示程序支持在首页设置临时的商户ID和密钥，无需修改配置文件：

1. 在首页的"商户配置"区域输入您的商户ID和密钥
2. 点击"保存配置"按钮应用临时配置
3. 临时配置将存储在会话中，优先于配置文件中的设置
4. 点击"重置为默认"按钮可以恢复使用配置文件中的设置

## 接口节点

平安夜支付接口提供了多个备用节点，以确保服务的可用性：

- 主节点: `https://pro.qjpay.icu`
- 备用节点1: `https://pro1.qjpay.icu`
- 备用节点2: `https://pro2.qjpay.icu`
- 备用节点3: `https://pro3.qjpay.icu`

## 功能特性

本演示程序实现了平安夜支付接口的以下功能：

1. **API创建订单**
   - 通过API接口创建支付订单
   - 获取支付二维码和支付链接
   - 适用于需要自定义支付页面的场景

2. **跳转支付**
   - 创建订单并跳转到平安夜收银台页面
   - 适用于不需要自定义支付页面的场景

3. **订单查询**
   - 查询订单的支付状态、金额等信息
   - 可用于检查订单是否已支付

4. **用户信息查询**
   - 查询商户的费率、余额等信息
   - 同时可查询指定订单的支付链接和二维码

5. **异步通知处理**
   - 处理支付平台发送的支付结果通知
   - 更新订单状态并执行相应业务逻辑

6. **同步跳转处理**
   - 处理用户支付完成后的页面跳转
   - 显示支付结果和订单信息

7. **接口文档**
   - 提供详细的API接口说明和签名算法详解
   - 包含请求参数、返回参数和示例代码

## 安全建议

1. 在生产环境中，务必将`config.php`放置在Web根目录之外，或做好访问限制
2. 商户密钥等敏感信息应妥善保管，避免泄露
3. 建议在服务器端实现更严格的参数验证和安全检查
4. 生产环境中建议使用HTTPS协议保证数据传输安全
5. 异步通知处理应做好防重复处理的措施

## 常见问题

1. **支付后没有收到异步通知**
   - 检查异步通知地址是否正确
   - 检查服务器是否能被外网访问
   - 检查防火墙设置
   - 检查`notify.php`是否有报错

2. **签名验证失败**
   - 检查商户密钥是否正确
   - 检查签名算法是否与文档一致
   - 检查参数格式是否正确

3. **订单查询不到**
   - 检查商户ID和密钥是否正确
   - 检查订单号是否正确
   - 检查API地址是否正确

4. **二维码无法显示**
   - 确保网络可以访问腾讯二维码生成API
   - 检查二维码内容是否正确
