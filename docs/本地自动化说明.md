# 本地自动化说明

这个模式下，整个执行链路都在你自己的电脑上完成，不依赖云端执行器。

## 1）安装依赖

```bat
pip install -r requirements_mobile.txt
```

## 2）启动本地桥接服务

```bat
python ??\local_bridge_api.py
```

默认地址：`http://127.0.0.1:9000`

## 3）启动下单服务

```bat
python ??\mobile_charge_server.py
```

打开方式：

- 下单页：`http://你的局域网IP:8000`
- 后台页：`http://你的局域网IP:8000/admin`

## 4）提交流程说明

1. 手机网页把订单提交给 `services/mobile_charge_server.py`。
2. 下单服务把任务发给本地桥接服务 `127.0.0.1:9000`。
3. 本地桥接服务调用 `services/local_charge_runner.py`。
4. 执行器返回 JSON 结果，订单状态变成 `SUCCESS` 或 `FAILED`。

## 5）本地执行器怎么接入

如果你要接自己的自动化方案，主要改的是：

- `services/local_charge_runner.py`

常见实现方式：

- ADB 或模拟器脚本
- AutoHotkey
- `pywinauto`
- Windows UIAutomation

执行器必须输出一行 JSON，例如：

```json
{"success": true, "message": "完成", "order_id": "local-123"}
```

## 6）可选环境变量

- `LOCAL_AUTOMATION_COMMAND`：桥接服务实际调用的命令
- `LOCAL_AUTOMATION_TIMEOUT`：执行超时秒数

示例：

```bat
set LOCAL_AUTOMATION_COMMAND=python ??\local_charge_runner.py
set LOCAL_AUTOMATION_TIMEOUT=180
python ??\local_bridge_api.py
```
