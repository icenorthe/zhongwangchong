# 手机下单服务说明

当前项目的手机下单方案主要由这两部分组成：

- `services/mobile_charge_server.py`：后端接口和订单处理逻辑
- `assets/web/mobile_order.html`：手机端下单页面

## 1）安装依赖

```bat
pip install -r requirements_mobile.txt
```

## 2）启动服务

```bat
python ??\mobile_charge_server.py
```

默认监听：`http://0.0.0.0:8000`

局域网访问地址：

- 下单页：`http://你的局域网IP:8000`
- 后台页：`http://你的局域网IP:8000/admin`

## 3）测试下单流程

1. 用手机打开下单页。
2. 提交设备号、插座号和金额。
3. 系统创建一条状态为 `PENDING` 的订单。
4. Worker 或本地 Agent 继续处理，最后变成 `SUCCESS` 或 `FAILED`。

支持 URL 预填参数，例如：

```text
http://<your-lan-ip>:8000/?station_name=29号站成都大学综合楼二号充电区V3.0&device_code=13061856473&socket_no=2&amount=1
```

## 4）配置官方接口

首次运行时，如果 `config/gateway_config.json` 不存在，系统会自动生成默认配置。

默认配置：

```json
{
  "mode": "mock",
  "base_url": "",
  "start_path": "/api/start-charge",
  "token": "",
  "headers": {},
  "extra_payload": {},
  "success_field": "success",
  "success_expected": true,
  "message_field": "message",
  "order_id_field": "order_id",
  "timeout_seconds": 15
}
```

切到正式模式时，可以改成：

```json
{
  "mode": "official",
  "base_url": "https://official-provider.example.com",
  "start_path": "/api/start-charge",
  "token": "replace-with-official-token",
  "headers": {
    "X-App-Id": "your-app-id"
  },
  "extra_payload": {
    "merchant_id": "your-merchant-id"
  },
  "success_field": "data.success",
  "success_expected": true,
  "message_field": "msg",
  "order_id_field": "data.orderNo",
  "timeout_seconds": 15
}
```

修改后可调用：

```bat
curl -X POST http://127.0.0.1:8000/api/admin/reload-gateway
```

如果设置了 `ADMIN_TOKEN`，记得在请求头里带上：

- `X-Admin-Token: <your-admin-token>`

也可以直接带 token 打开后台页：

- `http://<your-lan-ip>:8000/admin?token=<your-admin-token>`

失败订单重试：

```bat
curl -X POST "http://127.0.0.1:8000/api/admin/orders/<order_id>/retry?token=<your-admin-token>"
```

## 5）对外网开放时的建议

- 最好部署在带 HTTPS 的云服务器或反向代理后面。
- 建议用 Nginx 反代 Uvicorn。
- 后台接口用 `ADMIN_TOKEN` 限制访问。
- 在真正发起充电前，补上支付校验和订单校验。

## 6）打包 EXE（可选）

```bat
pip install pyinstaller
pyinstaller --onefile ??\mobile_charge_server.py
```

只打包服务进程还不够，仍然需要这些配套文件：

- `assets/web/mobile_order.html`
- `config/gateway_config.json`
- 可写的 `runtime/orders.db`
